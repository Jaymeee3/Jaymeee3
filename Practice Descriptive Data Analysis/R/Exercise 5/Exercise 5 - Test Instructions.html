The test consists of several tasks – complete as many as possible.
Complete the R section of the test in an Rmarkdown file. Submit the compiled Rmarkdown file in HTML format.

Task Information:

  • Some tasks may depend on the results of previous ones;
  • Copy the number and description of each task before starting, to clearly indicate which task you are working on;
  • The code must be commented, as the interpretation of the results is part of the evaluation. Comments are also useful because, even if the expected results are not achieved, it will be possible to assess how close the solution was.

R Section – Task List

  1. Import the dataset smartphone.csv (Note: the separator is a comma), which contains information about smartphones. Refer to the metadata. Create a copy of the dataset and name it smart.
  2. Clean up the dataset before starting. Rename variables that contain a period (e.g., price.USD.) so they are separated by an underscore and written in lowercase (Hint: you can rename them one by one using the rename function).
  3. Check that the data types of the imported variables have been correctly recognized, and explain which variables require type casting.
  4. Calculate the average weight of smartphones by brand (variables brand and weight.g.) and the standard deviation. Comment on the most interesting results.
  5. Based on the table created in task 4, graphically represent the average weight of smartphones by brand using a bar chart.
  6. Create a ranking of the 5 most expensive smartphones from the Huawei brand.
  7. It is assumed that whether a smartphone supports 8k video recording (variable video_8k) affects the final price (variable price.USD.). Represent the relationship between the variables and comment (Hint: video_8k is a categorical variable).
  8. Try to replicate the visualization using only smartphones that cost more than $500. Comment.
